About Quipucords

The Quipucords tool is designed to help users collect data about their usage of specific Red Hat software. By using Quipucords, users can reduce the amount of time and effort that is required to calculate and report usage of those Red Hat products.

What is Quipucords?

The Quipucords tool, also known as Quipucords, is an inspection and reporting tool. It is designed to find, identify, and report environment data, or facts, such as the number of physical and virtual systems on a network, their operating systems, and other configuration data. In addition, it is designed to find, identify, and report more detailed facts for some versions of key Red Hat packages and products for the IT resources in that network.

The ability to inspect the software and systems that are running on your network improves your ability to understand and report on your entitlement usage. Ultimately, this inspection and reporting process is part of the larger system administration task of managing your inventories.

The Quipucords tool requires the configuration of two basic structures to access IT resources and run the inspection process. A credential contains user access data, such as the user name and password or SSH key of a user with sufficient authority to run the inspection process on a particular source or some of the assets on that source. A source contains data about a single asset or multiple assets that are to be inspected. These assets can be physical machines, virtual machines, or containers, identified as host names, IP addresses, IP ranges, or subnets. These assets can also be a systems management solution such as vCenter Server or Red Hat Satellite Server.

You can save multiple credentials and sources to use with Quipucords in various combinations as you run inspection processes, or scans. When you have completed a scan, you can access these facts in the output as a collection of formatted data, or report, to review the results.

By default, the credentials and sources that are created during the use of Quipucords are encrypted in a database. The values are encrypted with AES-256 encryption. They are decrypted when the Quipucords server runs a scan with the use of a vault password to access the encrypted values that are stored in the database.

The Quipucords tool is an agentless inspection tool, so there is no need to install the tool on every source that is to be inspected. However, the system that Quipucords is installed on must have access to the systems to be discovered and inspected.

Is Quipucords right for me?

The Quipucords tool is intended to help you find and understand your Red Hat product inventory, including unknown product usage across complex networks. The reports generated by Quipucords are best understood through your partnership with a Red Hat Solution Architect (SA) or Technical Account Manager (TAM) or through the analysis and assistance supplied by the Subscription Education and Awareness Program (SEAP). Pilot programs that are currently underway include Quipucords as one tool that can help integrate your software inventory with other new and established Red Hat management offerings.

As an upstream project, Quipucords is not formally supported by Red Hat. However, there are certain customer scenarios and configurations where use of the upstream tool for scanning your environments and reporting your inventory is an ideal solution. If you are engaged with your SA, TAM, or a member of the SEAP team and have been instructed to use Quipucords as your inventory scanning tool, you will be supported by those groups as you install and use Quipucords.

Installing prerequisites for Quipucords

Before you begin the installation process, review the information about Quipucords prerequisites. Then complete any installation or configuration tasks for the prerequisites.

Procedure

Install the following requirements for hardware, software, and the environment in which you are going to install and use Quipucords.

Hardware prerequisites

The system on which you are going to install Quipucords must meet or exceed the following hardware requirements:

  • CPU: 2 core minimum, with a recommended 4 cores

  • Disk Storage: 30 GB

  • RAM: 1 GB minimum, with a recommended 2 GB

Software prerequisites

The system on which you are going to install Quipucords must have the following software requirements installed:

  • Operating system: The latest release of the following operating system versions:

    • Red Hat Enterprise Linux 6, 7, or 8

    • CentOS 6 or 7

      Note

      RHEL 7 or 8 or CentOS 7 are the recommended operating system versions. For RHEL 6 or CentOS 6, the minimum required kernel is 2.6.32-431 or later.

In addition to these software requirements, Quipucords has dependencies on other software. However, in some cases these dependencies are installed for you during installation, or instructions for their installation is integrated as part of the Quipucords installation process. The process to install dependencies can vary according several factors, including your operating system and whether you are doing an offline or online installation. Therefore, install the dependencies only as instructed by the Quipucords installation steps.

The Quipucords dependencies include the following software:

  • Ansible 2.4 or later, depending on the requirements of your operating system.

  • Container software, either Podman or Docker, depending on the requirements of your operating system.

  • The PostgreSQL database.

  • Python, 3.4 or 3.6, depending on the requirements of your operating system.

Other environment prerequisites

The environment in which you are going to use Quipucords must meet the following requirements. Some of these requirements affect the systems on which you are going to run Quipucords. Others affect the systems in your IT infrastructure that you are going to scan with Quipucords.

On the machine where Quipucords is installed:

  • If you install the Quipucords server and CLI packages on different machines, qpc-tools must be installed on both machines to run each installation.

  • If you install the Quipucords server and CLI packages on different machines, the CLI machine must be able to connect to the server machine.

  • The Quipucords server must have access to the IT infrastructure assets that are to be scanned.

On the systems where Quipucords runs scans:

  • Any network sources that are targeted for scanning must be running the Secure Shell (SSH) protocol.

  • A user account that is used as a credential for a scan requires the bash shell. The shell cannot be the /sbin/nologin shell or the /bin/false shell.

  • A user account that is used as a credential for a network scan must have adequate permissions to run commands and read certain files on those systems. For example, some commands that run during a scan require privilege elevation to gather the complete set of facts for the scan. The Using Quipucords guide has additional information about the creation of credentials for network scans and the privileges that must be associated with those credentials to enable a more complete scan of network assets.

  • A user account that is used as a credential for a network scan where authentication is done with an SSH key must have a copy of the private key on the Quipucords server. The private key must be stored in the ~/quipucords/server/volumes/sshkeys directory, the default location for this directory at the time of server installation.

Installing Quipucords with the online installation process

The online installation process runs an installer that uses Ansible to install the command line interface tool, the server image, and the database image.

When you run the online installation process, the server and CLI packages are installed with the default options. However, you can change some of the defaults used by the installation process by setting options that are defined in Options for the qpc-tools install command.

Prerequisites

  • Ensure that all prerequisites are installed and configured.

  • To use the online installation process, the machine on which you are downloading the installer and installing Quipucords must be connected to the Internet.

Procedure

To install Quipucords with the online installation process, you do the following tasks:

  1. Run the command to install qpc-tools. For more information, see Installing qpc-tools.

  2. Run the Quipucords installation with qpc-tools. For more information, see Installing with the online installation process.

    Note

    This installation process installs Quipucords with default settings for all of the options. However, other settings are available for you to customize your installation. For more information about these settings, see Options for the qpc-tools install command.

    • For RHEL 7 and CentOS 7, use the following command:

      # yum install -y https://github.com/quipucords/qpc-tools/releases/latest/download/qpc-tools.el7.noarch.rpm

    • For RHEL 6 and CentOS 6, use the following command:

      # yum install -y https://github.com/quipucords/qpc-tools/releases/latest/download/qpc-tools.el6.noarch.rpm

Installing with the online installation process

During the Quipucords installation process, you enter commands to install the server and the command line interface (CLI). The simplest method for installing Quipucords is to run the online installation with all of the default options. With this method, the install process prompts you to do the following actions:

  • Set a password for the Quipucords server administrator.

  • Set a password for the database user.

The following information includes commands for the default installation process. However, you can change some of the defaults used by the installation process by setting options that are defined in Options for the qpc-tools install command.

Procedure

To install the Quipucords server and CLI packages, use the following steps:

Installing the server

Install the server. The default server installation process prompts you to enter your user name and password for the Red Hat Container Catalog (the registry.redhat.io website). It also prompts you to supply a password for the Quipucords server administrator and a password for the PostgreSQL database user.

Important

As a best practice, note the Quipucords server administrator and the PostgreSQL database user passwords in the password management system that is used by your organization. The Quipucords tool does not offer a method to recover these passwords.

  1. Install the server by entering the following command:

    # qpc-tools server install

  2. At the prompt, set a password for the Quipucords server administrator.

  3. At the prompt, set a password for the PostgreSQL database user.

Verification steps

  1. For a successful installation of the server package, the output appears similar to the following example:

    Installation of the server was successful.

Installing the command line interface

Install the command line interface.

The recommended method is to include the options to set the port and IP address of the Quipucords server. These options must be set so that the Quipucords command line interface can communicate with the server.

  1. Install the command line interface by entering the following command, where server_port is the port that the Quipucords server is using for HTTPS communication and server_host is the IP address of the server:

    # qpc-tools cli install --server-port=server_port --server-host=server_host
Verification steps

  1. For a successful installation command line interface packages the output appears similar to the following example:

    Installation of the CLI was successful.

Options for the qpc-tools install command

The installation procedures in this guide install the Quipucords server and CLI packages with all of the default options. However, the install subcommand that installs the Quipucords server and command line interface packages includes options to customize the installation process.

The following information lists the install subcommand options, along with applicable usage information and default values, for the server and CLI packages.

Server installation options

--offline-files=_offline_files_path

Sets the fully qualified path to the files on the disconnected machine, for example, the transferred server and command line interface packages, that are needed to complete an offline installation. This option has no default value and is used only when specified during an offline installation.

--version=version.patch.release

Enables the installation of a specific Quipucords server version. Contains the semantic versioning format (version.release.patch, such as 0.9.0) of the Quipucords server that you want to install. This option has a default of the latest version. This option is required if the offline-files option is specified.

--home-dir=server_home_dir

Sets the fully qualified path to the installation directory for the Quipucords server. Defaults to ~/quipucords/.

--port=server_port

Sets the port number for the Quipucords server. Defaults to 9443.

--open-port=true|false

Determines whether to open the port in the firewall during the installation. This option enables communication between the Quipucords server and any remote clients over the port defined in the port option. Contains a true or false value. Defaults to true. Supply false to install without opening the server port in the firewall. The installation script must run with elevated privileges to open the server port.

--db-user=database_username

Sets the database user name for PostgreSQL. Defaults to postgres.

--db-password=database_password

Sets the database user password for PostgreSQL. This option has no default value. If omitted, qpc-tools prompts for a password.

--username=server_username

Sets the Quipucords server administrator user name. Defaults to admin.

--password=server_password

Sets the Quipucords server administrator password. This option has no default value. If omitted, qpc-tools prompts for a password.

Command line interface installation options

--offline-files=offline_files_path

Sets the fully qualified path to the files on the disconnected machine, for example, the transferred server and command line interface packages, that are needed to complete an offline installation. This option has no default value and is used only when specified during an offline installation.

--version=version.patch.release

Enables the installation of a specific Quipucords CLI version. Contains the semantic versioning format (version.release.patch, such as 0.9.0) of the Quipucords CLI that you want to install. This option has a default of the latest version. This option is required if the offline-files option is specified.

--home-dir=cli_home_dir

Sets the fully qualified path to the installation directory for the Quipucords CLI. Defaults to ~/quipucords/.

--server-host=server_IP_address

Sets the IP address to use to connect to the Quipucords server. This option has no default value. You must either set a value during the CLI package installation or set a value by configuring the server connection for the command line interface.

--server-port=server_port

Sets the port number to use to connect to the Quipucords server. This option has no default value. You must either set a value during the CLI package installation or set a value by configuring the server connection for the command line interface.

Installing Quipucords with the offline installation process

The offline installation process is required when you install Quipucords on a machine that does not have Internet access, also commonly referred to as an offline, air-gapped, or disconnected environment. The information in this section describes the installation process when you are installing the qpc-tools, the server, and the command line interface (CLI) tool in this type of environment.

When you run the offline installation process, the server and CLI packages are installed with the default options. However, you can change some of the defaults used by the installation process by setting options that are defined in Options for the qpc-tools install command.

For the offline installation process, you must complete some steps from a connected machine, one that has Internet connectivity. These steps include downloading the Red Hat Package Manager (RPM) package for qpc-tools, downloading the server and CLI packages, building a container image for the database, transferring packages to the disconnected machine, and others. You also complete steps on the disconnected machine, including installing dependencies to enable the offline installation, receiving the prepared packages and images transferred from the connected machine, and running the installer itself.

Prerequisites

  • Ensure that all prerequisites are installed and configured.

  • Ensure that the connected machine where you download packages can connect to the disconnected machine or machines where you want to install qpc-tools and the Quipucords server and CLI tool.

Procedure

To install Quipucords with the offline installation process, you do the following tasks:

  1. Install dependencies on the connected and disconnected machine, as required. These dependencies are additional required software that cannot be installed by qpc-tools in an offline installation process. For more information, see Installing dependencies for the offline installation.

  2. Download the qpc-tools package to the connected machine. For more information, see Downloading the qpc-tools package.

  3. Download the server package to the connected machine. For more information, see Downloading the server package.

  4. Build the database container image. For more information, see Building the database container image.

  5. Download the CLI package to the connected machine. For more information, see Downloading the command line interface package.

  6. Transfer the packages from the connected machine to the disconnected machine where you are going to run the installation. For more information, see Transferring packages to the offline machine.

  7. Run the installation. For more information, see Installing with the offline installation process.

    Note

    This installation process installs Quipucords with default settings for all of the options. However, other settings are available for you to customize your installation. For more information about these settings, see Options for the qpc-tools install command.

Installing dependencies for the offline installation

To use the offline installation process, you must install some dependencies manually. These dependencies are in addition to the prerequisites for Quipucords installation.

You must install the dependencies on the disconnected machine where you are installing and using Quipucords.

You must also install the applicable container software dependency on the connected machine where you are downloading the packages before installation. The container software (either Podman or Docker, depending on your RHEL or CentOS version) is required to be installed on the connected machine to build a container for the PostgreSQL database.

Procedure

Use one of the following procedures, as appropriate for your operating system.

Important

In the following procedures, a dependency might be duplicated in the qpc-tools, server, or command line interface sections. This duplication is intended to clarify which dependencies are required for these individual packages, for example, in a scenario where packages are installed on different machines. If you are installing all of the packages on the same machine, you are not required to install a dependency more than once.

Installing with the offline installation process

During the Quipucords installation process, you enter commands to install the server and the command line interface (CLI).

The following information includes commands for the default installation process. However, you can change some of the defaults used by the installation process by setting options that are defined in Options for the qpc-tools install command.

Procedure

To install Quipucords, you install three packages, the qpc-tools package, the Quipucords server package, and the Quipucords CLI package. Use the following steps:

Installing qpc-tools

Install qpc-tools.

  1. Run the following command, where /path is the full path to the location on the disconnected machine where you transferred the packages.

    # rpm -Uvh --force`/path`

Installing the server

Install the server. The default server installation process prompts you to supply a password for the Quipucords server administrator and a password for the PostgreSQL database user.

  1. Install the server by entering the following command, where /path is the full path to the location on the disconnected machine where you transferred the packages and version.release.patch is the version of the Quipucords server package:

    # qpc-tools server install --offline-files=/path --version=version.release.patch

  2. Enter a value for the Quipucords server administrator password.

  3. Enter a value for the PostgreSQL database password.

Installing the command line interface

Install the command line interface.

  1. Install the command line interface by entering the following command, where /path is the full path to the location on the disconnected machine where you transferred the packages:

    # qpc-tools cli install --offline-files=/path
Verification steps

  1. For a successful installation of the packages, the output appears similar to the following example:

    Installation complete.

Options for the qpc-tools install command

The installation procedures in this guide install the Quipucords server and CLI packages with all of the default options. However, the install subcommand that installs the Quipucords server and command line interface packages includes options to customize the installation process.

The following information lists the install subcommand options, along with applicable usage information and default values, for the server and CLI packages.

Server installation options

--offline-files=_offline_files_path

Sets the fully qualified path to the files on the disconnected machine, for example, the transferred server and command line interface packages, that are needed to complete an offline installation. This option has no default value and is used only when specified during an offline installation.

--version=version.patch.release

Enables the installation of a specific Quipucords server version. Contains the semantic versioning format (version.release.patch, such as 0.9.0) of the Quipucords server that you want to install. This option has a default of the latest version. This option is required if the offline-files option is specified.

--home-dir=server_home_dir

Sets the fully qualified path to the installation directory for the Quipucords server. Defaults to ~/quipucords/.

--port=server_port

Sets the port number for the Quipucords server. Defaults to 9443.

--open-port=true|false

Determines whether to open the port in the firewall during the installation. This option enables communication between the Quipucords server and any remote clients over the port defined in the port option. Contains a true or false value. Defaults to true. Supply false to install without opening the server port in the firewall. The installation script must run with elevated privileges to open the server port.

--db-user=database_username

Sets the database user name for PostgreSQL. Defaults to postgres.

--db-password=database_password

Sets the database user password for PostgreSQL. This option has no default value. If omitted, qpc-tools prompts for a password.

--username=server_username

Sets the Quipucords server administrator user name. Defaults to admin.

--password=server_password

Sets the Quipucords server administrator password. This option has no default value. If omitted, qpc-tools prompts for a password.

Command line interface installation options

--offline-files=offline_files_path

Sets the fully qualified path to the files on the disconnected machine, for example, the transferred server and command line interface packages, that are needed to complete an offline installation. This option has no default value and is used only when specified during an offline installation.

--version=version.patch.release

Enables the installation of a specific Quipucords CLI version. Contains the semantic versioning format (version.release.patch, such as 0.9.0) of the Quipucords CLI that you want to install. This option has a default of the latest version. This option is required if the offline-files option is specified.

--home-dir=cli_home_dir

Sets the fully qualified path to the installation directory for the Quipucords CLI. Defaults to ~/quipucords/.

--server-host=server_IP_address

Sets the IP address to use to connect to the Quipucords server. This option has no default value. You must either set a value during the CLI package installation or set a value by configuring the server connection for the command line interface.

--server-port=server_port

Sets the port number to use to connect to the Quipucords server. This option has no default value. You must either set a value during the CLI package installation or set a value by configuring the server connection for the command line interface.

Configuring and maintaining Quipucords

After installation is complete, you might have to complete other steps to configure or maintain Quipucords. The options that you choose during installation and the way in which you use Quipucords can determine the types of configuration and maintenance tasks that you need to perform.

Learn more

If you did not provide values to configure the connection between the Quipucords server and command line interface during installation, then you must configure those values before you can begin using Quipucords. To learn more, see the following information:

If you are going to run network scans with credentials that include SSH keys as the authentication method, then the Quipucords server must have access to the keyfile information. To learn more about adding SSH keys to the Quipucords server, see the following information:

Configuring Quipucords connections

The Quipucords command line interface must communicate with the Quipucords server through a specific port and IP address.

During the installation of the CLI package, you might have provided this information by setting values for the --server-port and --server-host options. If you did not provide those values during CLI package installation, you must configure those values. You can also use this procedure to edit these values, as needed.

Procedure

To configure the command line interface to communicate with the server, run the following command. If the server and CLI packages are installed on different machines, run this command from the machine where the CLI package is installed:

  1. Enter the following command, where server_port is the port that the Quipucords server is using for HTTPS communication server_host is the IP address of the server:

    # qpc server config --port=server_port --host=server_host

Adding SSH keys to the Quipucords server for network scans

When you configure sources and credentials for a network scan, you select the type of credential to use to authenticate to the network assets that are being scanned. One of the available options for the credential is to authenticate with a user name and SSH keyfile. If you choose this option, you must add a copy of the the private key to a specific directory on the server so that Quipucords can authenticate to those assets and complete the processes that occur during a scan.

You might have to perform these steps as an ongoing maintenance task as you create and refine the credentials needed for your network scans.

Procedure

To add an SSH keyfile to the Quipucords server:

  1. Copy the private key from the keyfile, using the copy method of your choice.

  2. Add the private key to the ~/quipucords/server/volumes/sshkeys directory on the Quipucords server, the default location for this directory at the time of server installation.

  3. Repeat these steps as needed for all credentials that use SSH keyfiles as the authentication method.

Accessing the Quipucords user interface

You access the Quipucords graphical user interface through a browser connection. You access the command line interface by running a command to connect to the server.

To use the Quipucords user interface, the system on which you want to run the user interface must be able to communicate with the system on which the Quipucords server is installed.

Logging in to the Quipucords user interface

Prerequisites

To log in to the Quipucords user interface, you need the IP address of the system where the Quipucords server is installed, the port number for the connection if the default port was changed during server installation, and the user name and password to use when logging in. If you do not have this information, contact the administrator who installed the Quipucords server.

Procedure

  1. In a browser, enter the URL for the Quipucords server in the following format: https://IPaddress:port, where IPaddress is the IP address of the Quipucords server and port is the exposed server port.

    The following examples show two different ways to enter the URL, based on the system that you are logging in from and whether the default port is used:

    • If you log in from the system where the server is installed and the default port is used, you can use the loopback address (also known as localhost) as the IP address, as shown in the following example:

      https://127.0.0.1:9443

    • If you log in from a system that is remote from the server, the server is running on the IP address 192.0.2.0, and the default port was changed during installation to 8443, you would log in as shown in the following example:

      https://192.0.2.0:8443

    After you enter the URL for the server, the Quipucords login page displays.

  2. On the login page, enter the user name and password and then click Log in to log in to the server.

Verification steps

If this is the first time that you have logged in to Quipucords, the Welcome page displays. You can begin by adding sources and credentials that can be used in scans. If you have previously logged in to Quipucords, the Welcome page is skipped and you can interact with your previously created sources, credentials, and scans.

Logging out of the Quipucords user interface

Procedure

  1. In the application toolbar, click the person icon or your user name.

  2. Click Logout.

Logging in to the Quipucords command line interface

Prerequisites

To log in to the Quipucords command line interface, you need the IP address of the system where the Quipucords server is installed and the port number for the connection. These values are set for the command line interface either during the installation of the CLI package or after installation as a configuration step. You also need the user name and password to use when logging in. If you do not have this information, contact the administrator who installed the Quipucords server.

The login command retrieves a token that is used for authentication with subsequent command line interface commands. That token is removed when you log out of the server, and expires daily.

Procedure

  1. To log in to the command line interface, enter the following command, where server_port is the port that the Quipucords server is using for HTTPS communication and server_host is the IP address of the server:

    # qpc server login --port=server_port --host=server_host

  2. At the prompts, enter the user name and password for Quipucords.

Logging out of the Quipucords command line interface

The command to log out of the server removes the token that was created when you logged in to the server. This token also expires daily.

Procedure

  1. To log out of the command line interface, enter the following command:

    # qpc server logout
